## Bug 1: Tracker issues

### Requirements (clarified by Lemz 2026-07-15):

1. **Tracker always visible on all chats.** The tracker section should always render, even when cleared/empty. Currently it hides entirely when title is blank and all answers are empty (`setTracker(null)` in PlayRoute).

2. **Blank title shows "Waiting" placeholder.** When the producer sends a blank title, the guest display should show "Waiting..." as the header (same as the existing waiting state). Don't hide the section.

3. **Pre-load on late join / refresh.** When a guest refreshes or joins late, the current tracker state (title + answers) should already be there. Currently `getRoster` handshake re-broadcasts roster + hostName but NOT the last tracker payload. Need to add tracker to the late-joiner sync flow.

### Implementation notes:

**Files to change:**
- `src/routes/PlayRoute.tsx`:
  - `trackerUpdate` handler: don't set `null` on empty - always keep tracker state, show placeholder when empty
  - `getRoster` handler on producer side needs to also re-send last tracker
  - Tracker display: when `tracker` is null or title is empty, show "Waiting on the answers for me to send" as the header, with empty answer slots
- `src/routes/ProducerRoute.tsx`:
  - `getRoster` response: include last tracker payload (`lastTrackerRef.current`)
  - Clear button already sends `title: ""` + empty answers (correct)
- `src/routes/ChatRoute.tsx`:
  - Check if tracker display exists on chats - if not, add it
  - Ensure late joiner sync covers this route too
- `src/lib/vdoninja.ts`:
  - `getRoster` response may need a tracker field added to the payload type

## Bug 2: Producer chat not pinned to bottom

The smart auto-scroll (only scroll when near bottom) was added in the latest staging commits. This is wrong for the live show use case. The producer (and all chat participants) should ALWAYS see the latest message, auto-scrolled to bottom, no matter where they've scrolled to.

**Current behavior:** If user scrolls up to read older messages, new messages don't auto-scroll. User has to manually scroll down.
**Expected:** Always pin to bottom on new message. No "smart" near-bottom check.

**Files to fix:**
- `src/routes/ChatRoute.tsx` - `ChatFeed` component, `useEffect` on `messages.length`
- `src/routes/PlayRoute.tsx` - `ChatPanel` component, same pattern

**Fix:** Remove the `nearBottom` check, always set `scrollTop = scrollHeight` on new messages.
