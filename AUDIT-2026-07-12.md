## 2. What was fixed (commit `2072871`)

### Live show bugs

**BUG-1 (Featured chat shows "Guest" after host refresh):**
Root cause: VDO.Ninja chat uses the `&label=` URL param as the author identity. When Sam refreshes, the label resets to VDO.Ninja's default. The roster name (set in producer panel) was never pushed to the VDO.Ninja iframe.

Fix: On `rosterUpdate`, if the guest's roster name differs from their current VDO.Ninja label, push the new name via `postMessage({ label: newName })` into the VDO.Ninja iframe. This keeps chat labels in sync with producer-set names. Also fires on `getRoster` reply (mount-time handshake), so late joiners and refreshers get corrected.

Also added: `featureMessage` in PlayRoute now resolves the VDO.Ninja label to a roster name before sending to the overlay, as a fallback when labels are stale.

**BUG-2 (STFU auto-unmute broken):**
Root cause: When STFU's 10s timer expired, the code cleared `stfuMutedRef` and called `reconcileMic()`. But `reconcileMic` only sent `postMessage({ mic: false })` when `stfuMutedRef` was true (the mute path). When STFU cleared, it fell through to the "not muted" branch — which never sent `mic: true` to actually unmute. The code assumed VDO.Ninja would unmute on its own when you stopped sending `mic: false`.

Fix: Added an `else if (!hostMutedRef.current)` branch in `reconcileMic` that explicitly sends `postMessage({ mic: true })` when STFU is cleared and the host hasn't separately muted the guest.

**IMPROVE-1 (Late joiner sync):**
- Underlay now loads roster from `localStorage` (`gamified.roster.v1`) as instant cache before `getRoster` reply arrives, instead of showing empty names.
- Underlay sends `getRoster` on mount (1.5s delay, same pattern as PlayRoute).
- Roster updates persist to `localStorage` on the underlay too.
- `hostName` also cached and persisted.

### Code review fixes

**§1.3 — `validatePayload` cardPlay targetSeat:**
Changed from `if (typeof p.targetSeat === "string" && !isValidSeatId(p.targetSeat)) return null;` to `if (!isValidSeatId(p.targetSeat)) return null;`. Now requires a valid seat ID rather than letting missing/undefined through.

**§1.4 — Buzz events accept `target` instead of `seat`:**
Removed `?? p.target` fallback from `buzzIn`/`buzzOff` validation. Now validates `p.seat` only, since no sender ever uses `target` for buzz events.

**§1.5 — Chat screen duration hardcoded in two places:**
`OverlayRoute.tsx` now uses `` `cardIn ${CHAT_SCREEN_MS}ms cubic-bezier(0.2,1.5,0.4,1) forwards` `` instead of literal `15s`. Both the expiry timeout and the CSS animation now reference the same constant.

**§1.6 — Empty tracker always renders "Waiting..." rows:**
Changed `tracker` state init from an object to `null`. The render gate `{tracker && (...)}` now hides the section until the first real `trackerUpdate` arrives. When the producer clears the tracker (empty title + all empty answers), state resets to `null` and the section hides again.

**§1.7 — Chat autoscroll fights the reader:**
Both PlayRoute `ChatPanel` and `ChatRoute` `ChatFeed` now check `el.scrollHeight - el.scrollTop - el.clientHeight < 60` before scrolling. If the user has scrolled up to read, new messages no longer yank them back down.

**§1.8 — Card announce React key collision:**
Changed `const id = \`ca-${Date.now()}\`` to use the existing `nextId()` counter function, preventing same-millisecond collisions.

**§1.9 — SFX self-skip asymmetric for host:**
Reviewed. VDO.Ninja's pcs (peer connection) does not reflect broadcasts back to the sender. No code change needed — confirmed behavior is correct as-is.

**§1.10 — Silent failure on chat send when iframe not mounted:**
Left as-is. Low priority, acceptable behavior (draft stays in input, user retries).

### Event protocol (VDO.Ninja data channel)

All events are P2P via VDO.Ninja's `sendData` API. Validated at runtime by `validatePayload()` in `vdoninja.ts`.

| Event | Direction | Payload |
|-------|-----------|---------|
| `rosterUpdate` | Producer → all | `{ names: Record<SeatId, string>, hostName?: string }` |
| `getRoster` | Any → Producer | `{ ts: number }` — request current roster |
| `cardPlay` | Guest/Host → all | `{ cardId, from: { seat, label, kind }, targetSeat, targetLabel?, ts }` |
| `cardReset` | Producer → all | `{ epoch: number }` — resets card use counts |
| `getResetEpoch` | Any → Producer | requests current epoch |
| `trackerUpdate` | Producer → all | `{ title: string, answers: Record<SeatId, string> }` |
| `emoji` | Guest → underlay | `{ emoji: string, seat: SeatId, ts: number }` |
| `buzzIn` / `buzzOff` | Guest → all | `{ seat: SeatId, ts: number }` |
| `muteGuest` / `unmuteGuest` | Host → target | `{ seat: SeatId, ts: number }` |
| `guestSelfUnmuted` | Guest → Host | `{ seat: SeatId, ts: number }` |
| `chatToScreen` | Host → overlay | `{ author: string, message: string, ts: number }` |
| `calibration` | Producer → underlay | `{ tiles: TileMap }` |

### STFU mute system

Dual-flag: `hostMutedRef` (advisory, guest can self-unmute) + `stfuMutedRef` (force-mute, 10s timer). Circuit breaker: 500ms interval re-asserts `mic: false` while either flag is true. When both clear, `reconcileMic` now explicitly sends `mic: true` to unmute.

### Late joiner / refresh sync

1. PlayRoute + UnderlayRoute send `getRoster` on mount (1.5s delay)
2. Producer replies with committed roster + hostName
3. PlayRoute pushes roster name to VDO.Ninja iframe as `{ label }` to keep chat labels synced
4. UnderlayRoute loads roster from localStorage as instant cache before reply
5. Roster + hostName persist to localStorage on every update

### Key constraints enforced in code

- No backend server (VDO.Ninja P2P data channels only)
- No new VDO.Ninja peer connections
- No `localStorage` for shared state (per-browser only, cache pattern explicit)
- Overlay/underlay are separate OBS browser sources (cross-source CSS filters don't work)
- Producer's Virtual Camera is what guests see (`roombitrate=0`)
- `postMessage` origin allow-list for VDO.Ninja domains
